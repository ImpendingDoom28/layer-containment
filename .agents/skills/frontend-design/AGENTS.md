# Frontend Design

Use this skill when the user asks to design, style, beautify, or build a frontend interface such as a page, component, dashboard, landing page, poster-like layout, or other web UI.

## Goal

Create distinctive, production-grade frontend interfaces that feel intentionally designed rather than generic.

## Core approach

Before coding:

1. Identify the interface purpose, audience, and constraints.
2. Choose a clear aesthetic direction and stay consistent.
3. Decide what makes the result memorable.

While implementing:

- Prioritize polished, working code over mockup-only ideas.
- Use strong typography, deliberate color systems, and cohesive spacing.
- Prefer bold, specific visual choices over safe default UI patterns.
- Add motion and visual detail only when they reinforce the chosen direction.
- Avoid generic AI-looking aesthetics, cookie-cutter layouts, and overused font/color combinations.

## Design prompts to answer internally

- What tone fits this interface?
- What visual choice will users remember?
- Does the layout feel intentional and specific to the brief?
- Are the styling, motion, and details all supporting the same concept?
